Sidecoach treats design work as a multi-phase pipeline rather than a single prompt. Each phase gets explicit guidance, the right references, and validation gates that fail loudly when something drifts.

Six taste failures the rule engine used to miss.

The taste validator inspects the HTML you actually produce. When a flow ships a fabricated icon, a translateY hover, or a hero radial blob, the orchestrator fails the gate and the output never reaches the user.

taste/fabricated-svg

Fabricated SVG icons

Any inline path without a verifiable library marker fails. Icons must come from one of eight approved libraries with verbatim path data.

taste/translatey-in-hover

translateY on hover

Hover-lift is an AI-product reflex. The tactical polish standard is scale on active, not vertical translation on hover.

taste/large-inline-style

Inline style bloat

More than fifty lines of CSS inside the head fails. Rules belong in the design system, not duplicated into the page.

taste/hero-radial-blob

Hero radial gradient

The radial-gradient hero blob is the most recognizable AI-slop signature. Heroes hold real photography, real typography, or real product.

taste/hex-in-interactive-state

Hex literals in hover

When a file defines CSS custom properties, the hover and active states must derive from them. Hard-coded hex skips dark mode and theming.

taste/border-radius-inconsistency

Radius inconsistency

More than two distinct radius literals means the page invented its own scale. Concentric radii derive from named tokens.
