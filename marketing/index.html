Core architecture

Five systems

Sidecoach layers five specialized systems on top of Impeccable to turn static analysis into a live design system that learns, enforces, and catches regressions.

Deterministic Validator

Hard gates that block flow execution when real prerequisites are unmet. Checks for PRODUCT.md content, DESIGN.md sections, design-md lint, tier dependencies, and motion library availability.

Phase 3 complete

FlowHistory v2

Cross-session memory keyed by project path, not session ID. Stores up to 20 runs per flow with full execution history for regression comparison and baseline tracking.

Phase 2 complete

RegressionDetector

Compares each flow's current output against prior runs. Detects status degradation, guidance count drops, checklist regression, and message quality loss. Blocks on severity.

Phase 4 complete

ProjectPersonaEngine

Generates 3 project-specific critique personas from your freeform PRODUCT.md via LLM extraction. Replaces Impeccable's 5 fixed generic archetypes with personas learned from your actual users.

Phase 5 complete

DesignDebtTracker

Auto-logs warning-level violations as formal design debt. Tracks deferral, estimates cost, and surfaces at session start. Close the loop on known-but-deferred issues.

Phase 6 complete

Impeccable Bridge

Runs npx impeccable detect as a sub-step in the multi-lens audit flow so the real 28-rule static analyzer fires during design audits, not just suggestions.

Orchestrated workflows

36 flows
5 tiers
Phase-gated

Flows build on each other. Tier 1 teaches the system. Tier 2 establishes design. Tier 3 polishes. Tier 4 verifies. Tier 5 ships. Intent detection chains them invisibly as you work.

Tier 1: Teach (Flows A-E)

Foundation. Establish PRODUCT.md and DESIGN.md as the source of truth.

  • Flow A: Teach product strategy (users, brand, anti-references)
  • Flow B: Teach design tokens (colors, typography, scale)
  • Flow C: Validate PRODUCT.md against schema
  • Flow D: Validate DESIGN.md against Google spec
  • Flow E: Lint DESIGN.md with npx @google/design.md

Tier 2: Build (Flows F-I)

Requires: Tier 1 complete. Craft features with design as the driver.

  • Flow F: Shape feature (plan UX/UI before code)
  • Flow G: Craft feature (build end-to-end)
  • Flow H: Extract tokens (pull patterns into system)
  • Flow I: Document generated DESIGN.md

Tier 3: Polish (Flows J-P)

Requires: Tier 2 complete. Refine typography, motion, color, and craft.

  • Flow J: Typeset (improve hierarchy and readability)
  • Flow K: Audit multi-lens (static + impeccable + a11y)
  • Flow L: Critique Nielsen + project personas
  • Flow M: Polish (final alignment against design system)
  • Flow N: Animate (add purposeful motion)
  • Flow O: Colorize (strategic color application)
  • Flow P: Delight (personality and memorable touches)

Tier 4: Verify (Flows Q-T)

Requires: Tier 3 complete. Production-ready sweep.

  • Flow Q: Harden (errors, i18n, edge cases)
  • Flow R: Adapt (responsive design across devices)
  • Flow S: Optimize (performance and rendering)
  • Flow T: Onboard (first-run flows, empty states)

Tier 5: Master (Flows U-V)

Requires: All tiers complete. System-level operations.

  • Flow U: Reflect (corpus analysis - patterns, tensions, gaps)
  • Flow V: Architect (multi-feature orchestration)
