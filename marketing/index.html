The Yes& Claude Code stack

Working with Claude is fast.
Working with Claude well is a discipline problem.

The model forgets between sessions, has no opinions of its own, and will cheerfully ship a half-verified fix unless something stops it. Most "AI workflow" advice solves this by adding ceremony, which the model promptly skips under pressure. We took a different bet. 

curl -fsSL https://raw.githubusercontent.com/jonahscohen/claude-dotfiles/main/bootstrap.sh | bash

One curl. One shortcut. Twelve components, all additive, all undoable.

The bet

Externalize the discipline into files the harness enforces. Rules in CLAUDE.md, loaded every session. Hooks in settings.json that block bad behavior at the tool boundary. Memory in .claude/memory/ that persists across machines, across teammates, across time.

The result isn't a productivity hack. It's a way of working with an AI colleague that has memory, opinions, and a verification protocol. Twelve components, four houses of thought, one curl.

What's inside

The four houses

Each house answers a question the team had to solve before Claude could be trusted with real work. The houses are the shape of the system. Everything else is implementation.

01

Discipline

The system stops Claude from shipping work it hasn't verified. Rules in a file are advisory. Hooks at the tool boundary are mechanical. We chose mechanical.

  • Refuses commits, edits, and tools that break the rules
  • Gates "done" behind real verification, not "looks right"
  • Each rule traces back to a failure we'd rather not repeat

02

Memory

Claude becomes a colleague who remembers what was decided last week, and why. Notes live in your repo, ride along with git, and load at the start of every session.

  • Per-task memory, not per-session - granularity matters
  • Indexed, structured, attributable - your name on your work
  • Audited periodically by a reflection layer that finds drift

03

Design

Generated UI lands on-brand because the work routes through a design pipeline, not a vague prompt. Specialists for typography, motion, references, polish. A generalist that spans all of them.

  • Strategy reads your PRODUCT.md before deciding anything
  • Typography refuses training-data defaults like Inter and Fraunces
  • A personal catalog of patterns you've seen in the wild

04

Workflow

One curl, one shortcut, all additive. Boost an existing Claude Code without losing your config. Pick any subset; the rest is a no-op. Re-runs are safe. Uninstall is clean.

  • Twelve components in a checkbox TUI - take defaults, hit enter
  • Marker-guarded - removes exactly what was installed, nothing more
  • Works alongside your existing CLAUDE.md and settings.json

How we know it works

Built on itself.

Every layer in the stack traces back to a specific failure we wanted to stop repeating. Nothing in here is a feature in search of a problem.

  1. The reflection layer

    The memory corpus got large enough that drift was inevitable. Five parallel agents now audit the corpus on demand and surface what's emerging, contradicting, or going stale.

    It exists because the corpus needed it.

  2. The personal catalog

    Public catalogs index components. They have no home for "the way Linear's inbox staggers" or "the keycap treatment Unlumen uses." So we built a capture wizard for the patterns you actually see in the wild.

    It exists because the patterns worth remembering didn't fit anywhere else.

  3. The second-fix gate

    A regression shipped because a second fix landed on the same file before the first one was verified. Now a hook warns when that pattern repeats, while staying silent on legitimate multi-file changes.

    It exists because the failure is too easy to repeat without it.

The package

Twelve components.
Pick any combination.

Defaults are all on, so most teams just take them. Every component is additive, marker-guarded, and undoable.

brain

Team rules and shared workflow, appended to your CLAUDE.md between markers.

config

Safety hooks, enabled plugins, and permission patterns merged into your settings.json.

memory

Memory discipline rules, three lifecycle hooks, and the session loader script.

skills

The design pipeline plus four peer skills and the personal-catalog seed.

statusline

A custom prompt-bar render that shows session and memory state at a glance.

cmux

Split-pane terminal config that drives the in-app browser preview.

nvm

Fix for "claude: command not found" in fresh terminals when nvm doesn't auto-activate.

ampersand

The shortcut you actually use: re-launch the installer from any directory.

discord

A Discord chat agent launcher with cold/mid/warm onboarding states.

voice-input

Local voice-to-text via whisper.cpp and ffmpeg. No cloud, no API key.

voice-output

Spoken summaries via OpenAI TTS. Starts muted. Three ways to toggle.

reflect

Memory corpus analysis. Five agents look for patterns, tensions, gaps.

Get the stack

One curl.
Twelve components.
Your discipline, installed.

curl -fsSL https://raw.githubusercontent.com/jonahscohen/claude-dotfiles/main/bootstrap.sh | bash
  • Already running Claude Code? Every component is additive. ampersand --only brain,config,memory takes the rules, hooks, and memory subsystem without touching the rest of your config.
  • Fresh Mac? Run the curl, take TUI defaults, open a new terminal. You're set up.
  • Want to undo? Marker-guarded blocks in your CLAUDE.md and settings.json. sed removes exactly what we added.
