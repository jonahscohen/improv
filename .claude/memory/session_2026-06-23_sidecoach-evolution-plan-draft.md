## SIMPLICITY CLAIM, STATED HONESTLY (lead gate critique, non-negotiable)
Three distinct claims, never conflated:
1. SIMPLER THAN CURRENT SIDECOACH - true and large (~42k lines / 142 src + 42 MCP files / 6 routing impls / triplicated classifier / 4 vocabularies -> the BILL OF MATERIALS below). Claim this loudly; it is the core of "net-simplify".
2. CAPABILITY vs ORACLE = a clean SUPERSET, not "smaller". Reimplementing the detector correctness floor (from public specs) + adding our idiom rules is deliberately MORE on the detection axis. We do not claim "smaller than oracle on capability"; we claim "everything oracle detects (proven by the oracle eval), plus our idioms + drift + memory + references + convergence" - all OUR code.
3. USER SURFACE vs ORACLE = EQUAL-OR-SIMPLER (one verb vocabulary + NL->verbs + one `scan` primitive vs oracle's 23 commands + smart menu). This is where the "simpler than oracle" claim is honest and defensible.
We win by being a clean superset with an equal-or-simpler surface and PROVEN outcome superiority - NOT by being smaller than oracle. Any plan sentence that implies "simpler than oracle in every way" is an overclaim to be struck.

SUPERIORITY FRAMING - PARITY vs BETTER (concern 6; JONAH-CONFIRMED 2026-06-23, no longer pending): you cannot "beat" correctness - correct contrast is correct contrast. Jonah's ruling, the net claim = "NEVER WORSE ANYWHERE, DECISIVELY BETTER WHERE IT COUNTS." Operationalized as TWO gate types:
- PARITY / NON-REGRESSION (`>=` gates) on the correctness-floor axes (detector recall, precision/FP, contrast/OKLCH, blocking accuracy) + EXTENSION CAPACITY (catching idiom/drift classes oracle has zero coverage of). Detector parity is NOT a "beat"; it is "non-regression + extension capacity."
- STRICTLY BETTER (strict-`>` gates) on the differentiators: taste/guidance QUALITY, workflow, maintainability, distributability, protected leads (beats memory, references, crash-safe convergence). Sub-distinction by gate kind: the SAMPLED differentiator (taste quality, which has judging variance) requires STRICT-> with STATISTICAL SIGNIFICANCE (Contract 6 A5 paired CI); the DETERMINISTIC differentiators (workflow/maintainability/distributability/leads - facts, not samples) require STRICT-> by a deterministic test (existence/metric), where statistical significance is N/A (you don't bootstrap "has beats memory"). Honesty note: claiming "statistical significance" on a deterministic capability would be a category error; the rigor there is a pass/fail test, not a p-value.

- A2 PRECISION / FP-RATE on known-good (precision >= oracle AND >= 0.85; FP-rate <= oracle AND <= 0.10 false findings/page) [round-14: must be >= oracle, not just a local floor]. Every Sidecoach-extension rule ships known-good negatives + a suppression/appeal path (extension rules can be noisy/taste-authoritarian). "More rules" is NOT a beat unless precision does NOT regress.
- A3 contrast/OKLCH floor (|Sidecoach - oracle WCAG ratio| <= 0.01, exact).
- A4 BLOCKING accuracy on a labeled block/non-block corpus (decision accuracy >= oracle AND >= 0.95; misblock <= oracle AND <= 2.5%; severity-mismatch <= oracle AND <= 5%) [round-14: every sub-metric must be >= / <= oracle, not just a local floor].
  -> The detector "win" is honestly NON-REGRESSION on A1-A4 + EXTENSION CAPACITY (catches idiom/drift classes oracle has ZERO coverage of), not a correctness "beat".
STRICTLY-BETTER axis (the differentiator):

STATISTICAL VALIDITY (concern 4 + round-10 + re-gate round-3 concern 2 - remove post-hoc knobs AND power-lock N): PAIRED per-brief comparison. n >= 20 is a FLOOR, not adequate power for a 0-20 subjective rubric; in Stage 0 a PRE-REGISTERED POWER ANALYSIS (from pilot/calibration variance) LOCKS the actual N before any scored output. The win-rate alternative is REMOVED (gameable post-hoc knob). GUARDRAIL: zero SEVERE LOSSES = any brief where Sidecoach composite is >= 3 points (>= 15% of the 20-pt scale) below oracle.
