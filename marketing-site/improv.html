visual / improv

Stop describing the UI. Show it.

improv is the editor that watches your live UI. Select a region in the browser, tweak the values in the side panel, see the change immediately. When the tenth round-trip of "make it a little tighter" is in your future, this is the off-ramp.

What it does

Direct manipulation, not synthesized clicks.

The two-pane mode pairs a live preview with a property inspector. Box model, fill, border, shadow, opacity, text alignment - every adjustment goes through the same code path your dev server sees.

Watch a region.

Click an element in the preview. improv lights up its selector path, opens the property panel, and pins focus on the values that matter for that element type.

No DOM probing on the validation side - you click real buttons, change real inputs, see the real result.

Change a value.

Drag a number, pick a color, swap an icon. Every control writes to a pending-changes buffer the dev pipeline can read.

Pending changes preview live. Apply commits them to your source files. Discard reverts.

Loop it.

improv keeps a queue of in-flight prompts. When you ask Claude to "make the heading bigger" while watching the preview, the response and the visible change land at the same time.

The watch loop reconnects through context compactions. If a session restarts, the queue and selection survive.

How to start

A single install step, then open a file.

improv is one component of the dotfiles. Install it standalone or as part of the full toolkit. 

./install.sh --only improv

The installer wires the editor extension, the MCP server, and the watch loop. Open a project, run improv start, and the preview pane appears in your browser.

Posture

Things improv refuses to do.

The shape of the tool is defined as much by what it skips as what it does.

No synthetic events.

improv interactions are real clicks and real keypresses against the real DOM. The validation guard blocks element.click(), dispatchEvent, private-API access, and getComputedStyle probing.

If the click can't reach what you need, that's a signal to widen the selector or fix the structure - not to reach in through JavaScript.

No design system imposed.

improv reads your project's tokens. If you have a DESIGN.md, those values are the dropdown options. If you don't, improv falls back to free input and complains in the console.

No remote.

The editor runs in your browser against your dev server. There is no cloud render, no shared state, no preview link to mail around. If you want to share, take a screenshot the same way you would for anything else.
