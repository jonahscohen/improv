design / sidecoach

Say what you want. Sidecoach puts it all together.

Sidecoach is the design layer of Improv. You describe the task the way you would to a colleague, it recognizes the kind of design work you mean, runs it end to end, and validates the work against your brand, taste rules, and accessibility guidelines.

How it Works

All you need to do is tell Claude what to do.

You say what you want in plain language. Sidecoach works out which pass it needs, then runs that pass from start to finish.

Setting It Up

Two files, and every pass is measured against your brand.

Sidecoach does its best work when it knows your product. It reads two files from your project root, and you create them once, by asking - not from a blank page.

PRODUCT.md

Who the product is for, the brand voice, and what it should not look like. The strategic context: register, users, personality, and the patterns to avoid.

Say teach Sidecoach about this project. It interviews you for the gaps, reads anything you already told it, and writes the file. For a brand-new project it goes deeper and flags vague answers like modern so you sharpen them.

DESIGN.md

The design tokens: colors, typography, spacing, rounding, and component specs. The visual source of truth, written to the open design.md spec so generated code references tokens by name instead of pasting hex values.

Say document the design system. If the project already has CSS, Sidecoach scans it for tokens and writes the file for you.

The everyday workflow

Build something. Then have it reviewed and polished.

The rhythm is simple, and you do both in plain language. The quality checks run as part of the pass, so nothing ships half-checked and you never have to remember to ask for them.

Straight to the point

Plain language is the default. The verbs are the shortcut.

Know the pass you want? Name it as a command: /sidecoach polish, /sidecoach audit, /sidecoach craft. Twenty-two verbs, grouped by phase.

Plan

  • shape
  • onboard

Plan a feature's design before any code; design first-run and empty states.

Build

  • craft
  • layout
  • typeset
  • animate
  • colorize
  • clarify
  • delight
  • bolder
  • overdrive

Build a component from scratch, restructure layout, set type, add motion or color, sharpen the message, or push the visual weight up.

Review

  • audit
  • critique
  • polish
  • harden
  • adapt
  • optimize

Technical audit, design critique, tactical polish, production hardening, a responsive pass, and a performance pass.

Tone

  • quieter
  • distill

Reduce noise and complexity, or strip a design down to its essentials.

Document and live

  • document
  • extract
  • live

Generate DESIGN.md from existing CSS, pull reusable tokens out of an implementation, or iterate on a target in the browser in real time.

What you get back

A build report, graded by area, backed by real checks.

Every pass ends with a readable report, not a wall of logs. It grades the work area by area, so a strong overall result never hides one weak spot.

Graded per area.

Accessibility, theming, responsiveness, motion, and craft each get their own mark, so you can see at a glance where the result is strong and where it needs another look.

Backed by real checks.

The grades come from actual tests: contrast measured against WCAG, tap targets measured in pixels, tokens checked for completeness, not a model's impression that it looks fine. Sidecoach can open your real rendered page in a browser and catch problems there, and that check runs server-side, so it works whether you are on the command line, the desktop app, or your phone.

It remembers the work.

When a pass finishes, Sidecoach records what it did to your project's Beats, the running memory of the project, so the next session knows what was built, what was decided, and why, instead of starting cold.

Why it refuses shortcuts

Skipping the gates is how AI work ends up looking like AI work.

Each standard Sidecoach holds corresponds to a failure mode we have watched language models produce on real projects. The pass does not finish around them.

Real icons only.

Icon artwork comes verbatim from an established library: Heroicons, Lucide, Tabler, Phosphor. Nothing hand-drawn or approximated reaches your page, and a project is held to one library so the set stays consistent.

Live tokens, not hard-coded colors.

Generated interface code points at your real DESIGN.md tokens. A hex literal that did not come from your tokens is a partial grade, never a pass, so the system never quietly drifts.

No template phrases.

The critique watches for the category reflexes: "request a demo," "we use AI to do X," gradient-mesh heroes, carousels of fake dashboards. Findings get a severity tag and a fix, not a shrug.

No silent skips.

The work does not half-finish. If something serious fails, you get a named blocker in the build report under that area's grade, not a quiet pass that looks done and is not.

How to start

Install it, teach it your project, then describe the work.

Sidecoach is one piece of Improv. Install it on its own or as part of the full toolkit, set up the two files once, and start in plain language.
