design / sidecoach

A design orchestrator that refuses to skip the work.

Eight craft flows, 22 verb commands, 159 validators across ten design domains, one chain that runs from PRODUCT.md to the polish checklist without losing the thread.

The chain

Eight flows. Each one earns the next one.

When you run /sidecoach craft against a project, sidecoach walks the chain in dependency order. Skip nothing. Each flow's output gets validated and becomes input for the next.

  1. flowA

    Brand verification

    Reads PRODUCT.md. Resolves register (brand vs product). Caches the design laws.

  2. flowB

    Component research

    Pulls component patterns vetted against AI-slop heuristics. 11 interaction rules. 8 writing rules.

  3. flowE

    Motion patterns

    Researches easing curves. Exponential-only - no bounce, no elastic. Six reduced-motion strategies.

  4. flowF

    Design tokens

    Validates DESIGN.md across all seven design domains. Twenty sections checked against the rule library.

  5. flowG

    Component implementation

    Generates token-backed component specs. Eight interaction states. Semantic copy. ARIA per state.

  6. flowH

    Motion integration

    Wires motion into components. Stack-aware templates. Exponential easing validated. Performance constraints checked.

  7. flowI

    Accessibility

    Seven WCAG 2.1 AA domain audits. Screen reader testing plan. Severity-prioritized findings.

  8. flowJ

    Tactical polish

    Sixteen-point refinement pass. Concentric radius. Scale on press. Tabular nums. Text balance. The details that separate good from shipped.

Verb commands

Twenty-two slash commands, every one earned by a flow.

The full verb surface, twenty-two commands. craft runs the full chain. polish runs just the refinement pass. audit, critique, shape, typeset, layout, and the rest each map to a focused flow or composite. 

/sidecoach craft marketing-site
/sidecoach polish src/components/Button.tsx
/sidecoach audit pages/checkout

The full command list is documented in SKILL.md. Run /sidecoach help <verb> for parity-checklist detail on any of them.

Why it exists

Because skipping the gates is how AI work ends up looking like AI work.

Every shortcut sidecoach refuses corresponds to a failure mode we've watched LLMs produce on real projects.

No fabricated icons.

SVG path data must come byte-for-byte from a real icon library - Heroicons, Lucide, Tabler, Phosphor. The reference system caches canonical path strings so flowG can verify them against what gets written into your codebase.

The ClaudemdMandate validator blocks the commit if a path doesn't match a registered source.

No hardcoded colors.

Every CSS value must resolve through a DESIGN.md token. flowF parses your tokens, flowG emits code that references them, and the build report grades each domain on token coverage.

A hex literal that didn't come from DESIGN.md is a partial grade, never a pass.

No "request a demo".

The category-reflex detector in flowD watches for AI-template phrases - "request a demo," "we use AI to do X," "powered by," gradient-mesh hero, screenshot-carousel-of-fake-dashboards. Findings get a severity tag and a fix recommendation in the build report.

No silent failures.

The chain doesn't half-finish. If a prerequisite is missing, you get a structured error naming the missing flow, not a silent skip. If a flow soft-fails, it surfaces in the build report under that domain's letter grade.

How to start

Install once. Add PRODUCT.md and DESIGN.md to your project.

sidecoach is part of the dotfiles. If you've cloned the repo, it's already installed. 

./install.sh --only sidecoach
echo "## Register\n\n**Brand**\n\n## Primary Users\n..." > PRODUCT.md
/sidecoach craft .
