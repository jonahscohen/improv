Modes - 6 shape-of-work keywords.

Modes name the shape of the work itself, not a single step. Each mode is a curated chain of verbs and takes precedence over verb matches in the same prompt. Type a mode word in any prompt to fire the chain.

Section 3

How a verb routes to a flow.

Two layers converge when you type a slash command. One layer recognizes intent. The other layer turns intent into a deterministic chain.

The keyword hook.

The UserPromptSubmit hook at claude/hooks/sidecoach-keyword.sh reads the prompt, sanitizes it (fenced code blocks, inline backticks, URLs, XML tag bodies, and transcript markers are stripped), then matches each verb in sidecoach-verbs.json with hyphen-aware word boundaries.

The first matching verb wins. Registry order breaks ties. Informational framings like "what is X" or "how do I X" suppress the fire. On a hit, the hook injects an additionalContext line naming the verb so the model routes to the right flow chain.

The verb registry.

The slash-command router in sidecoach/src/slash-command-router.ts looks up the verb in verb-command-registry.ts, retrieves the flowIds chain for that verb, and runs each flow in order.

Each flow produces guidance, a checklist, artifacts, and a memory entry. The orchestrator appends the verb-specific guidanceAppend lines after the chain finishes so the response speaks the verb's voice while keeping sidecoach's validators, BuildReport, taste validation, and memory infrastructure intact.

Why both layers exist.

The keyword hook ensures the right intent gets recognized even when the user does not type the slash command directly. The verb registry turns that recognized intent into a deterministic chain of flows.

Together they replace the older "skill auto-trigger" model that did not fire reliably on real builds. The keyword layer is the new detection surface; the flow chain is the new execution surface.
