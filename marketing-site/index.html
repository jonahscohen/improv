The toolkit

Three tools that earn their place. A handful more in the wings.

Every entry is something we use day to day. Equal billing - no hero tool, no consolation prize. Pick the one that maps to your problem and ignore the rest.

visual

improv

A two-pane editor that watches your live UI. Pick a region, change a value, see the result. The point is to short-circuit the screenshot-and-describe loop when you and Claude are nudging the same component for the tenth time.

See improv -> design

sidecoach

An opinionated design orchestrator. Eight flows, 159 validators, one chain from brief to polish. If you have a DESIGN.md, sidecoach will hold it to you.

See sidecoach -> continuity

memory

A layered memory system Claude writes to per task and reads on every session start. Survives compactions, forks across machines through the dotfiles repo, indexed by a single MEMORY.md you actually read.

In the wings

Smaller things that earn their keep.

The dotfiles ship a handful of supporting components - some hooks, some MCP servers, some shell glue. They're not pitched, they're documented in the README. Naming them here so you know they exist.

  • cmux Multi-pane terminal surface. The browser pane is the visual-verification path for any UI work.
  • voice Whisper.cpp transcription plus OpenAI TTS. Discord voice memos in, voice notes back.
  • discord A bot bridge so you can DM Claude from your phone. Per-user pairing, no group chats.
  • hooks A dozen pre- and post-tool hooks that enforce the team-rules: no emojis, no AI attribution, no skipped verification.
  • validation-guard Blocks synthetic clicks and DOM probing during verification. Real input only.

Posture

What this is not.

Worth being explicit. There are categories of "developer tool" that we're deliberately not.

Lives in your home directory.

There is no account, no dashboard, no telemetry. The whole thing is a folder you clone into ~/.claude. Uninstall is rm -rf.

If you fork it, your changes stay yours. Nothing phones home.

Components stand alone.

The components are independent. Install improv without sidecoach. Disable voice if you don't want the TTS bill. The installer breaks each piece out so you can opt in component by component.

A personal workstation setup.

This is a personal workstation setup that grew up enough to share. Opinionated about things you may disagree with. Read the rules at CLAUDE.md before you decide.
