visual / justify

Tweak the page. Claude writes the code.

Justify lets you change a live web page by pointing at it. Open your running site, click the thing you want different, say what should change in plain English, and hand the queue to Claude. It edits your actual source files and the page refreshes itself.

The one move

You tweak in the browser. Claude writes the code.

No copying selectors, no describing locations over chat, no trying to explain which button you mean. The page you are looking at is the interface, and the changes you make land as real edits to your files, not throwaway previews.

A toolbar over your own site.

Justify runs as a small toolbar layered over your running site. It does not replace your dev setup or take over the page - it loads one script alongside whatever you are already building and stays out of the way until you summon it.

For refining, not strategizing.

Justify is deliberately narrow: nudging spacing, swapping a label, fixing a color, raising a background that reads too faint. For larger work that needs strategy and research, reach for Sidecoach, which runs the broader design flow and hands live page tweaks back to Justify.

Two modes

Point and describe, or point and adjust.

There are two ways to change something, and you pick based on whether you know exactly what you want or just know it needs to be different.

Prompt mode.

Press p, click the element, and a small input appears right on it. Type what you want in normal language and press Enter to add it to the queue. This is the mode when you know the intent but not the exact CSS - you are telling Claude what to do, not how.

Real examples, each typed straight onto the element: "make this caret into an actual icon from our library," "this background layer is practically invisible, raise it," "max-width this paragraph to 75%, full width below that."

Manipulate mode.

When you know the exact property, click an element and scrub its value directly - drag a number, change a color, adjust spacing or radius. The page updates as you go, and the changes buffer until you send them. This is for precise, known adjustments: "this is 16px and should be 20px," not "this feels cramped." The toolbar reads your DESIGN.md tokens and surfaces them as the options.

Hand off and review

Stack the queue, hand it off, read the real diff.

Offers stack as you go - stack five, stack twenty. When the queue is ready, hand it to Claude: hit Send All in the toolbar, or say "watch justify" in chat. Claude works your real source in the background and the page hot-refreshes when the scene is done.

Watch the bar move.

A status bar tracks the handoff through three stages, so you always know where a change is. Working: Claude has claimed the task and is editing your source. Validating: it checks the change against your live page, the same page you are looking at. Review: the edit is done and the page hot-refreshes on its own, in every open tab.

Read the diff in the Changes panel.

The bottom-left Changes panel fills with a real before/after diff - a standard unified diff, old and new line numbers down the side, red and green +/- lines, exactly as you would see it in any code review. This is the actual edit to your files, not a paraphrase.

Each entry shows the file name, the size of the change (for example +583 -71), and an Open With control that jumps straight into your editor at the exact line that changed. Every entry also carries Revert to undo it and Reply to send Claude a follow-up.

When something is off

A few things go wrong predictably. Each has a quick fix.

The toolbar never appears.

The script did not load. Hard-reload first. If it still does not show, re-run justify-init <project-root> and confirm your site is served over http. An https-only site needs the secure core and a one-time trusted certificate.

You send changes but nothing happens.

Claude is not listening yet. Say "watch justify" in chat to start the loop, then hit Send All again. If the bar drifts to "Connected," the loop dropped - saying "watch justify" re-arms it.

A command reports the service is down.

The background server stopped. Run justify-serve to bring it back. It is safe to run anytime.

How to start

One command starts everything.

Justify is one piece of Improv. Install it once, then in Claude say /justify - that single command starts the background service, wires the script into your project, and switches the toolbar on. If your site is already running, you are tweaking within a minute. 

ampersand --only justify
# then, in Claude:
/justify

If you would rather do it by hand, it is three plain commands: justify-serve starts the service on port 9223, justify-init <project-root> wires the script into your stack, and opening your site over http brings up the toolbar. Toggle it any time with CMD+SHIFT+..
