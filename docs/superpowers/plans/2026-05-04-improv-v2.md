# Improv v2 Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development.

**Goal:** Rebuild Improv's UI layer to match Retune (manipulate) and Agentation (annotate/layout), fix activation model, add Apply workflow everywhere.

**Architecture:** Rewrite core/toolbar.ts (4 modes), core/manipulate/ (Retune-quality panel with element handles and box model), core/annotate/ (Agentation-quality markers), core/layout/ (Agentation-quality palette). Add improv init CLI for per-project activation.

**Tech Stack:** TypeScript, esbuild, DOM APIs (no innerHTML)

---

## Tasks

### Task 1: improv init CLI for per-project activation
Create improv/cli/init.ts that detects project stack and adds script include. Detect: Drupal (*.info.yml), WordPress (wp-config.php), Next.js (next.config.*), Vite (vite.config.*), generic HTML. Write appropriate dev-only script tag. Create .improv marker. Also create improv/cli/remove.ts to undo. Wire into install.sh as `~/.claude/improv-init` symlink.

### Task 2: Auto-activation on script load
Modify core/index.ts so when the script loads, it immediately shows the toolbar (no separate activate step). CMD+SHIFT+. toggles visibility. Remove the old "wait for activate message" pattern. The toolbar should appear on every dev page load.

### Task 3: Split toolbar into 4 modes
Modify core/types.ts: ImprovMode = 'manipulate' | 'prompt' | 'annotate' | 'layout'. Rewrite core/toolbar.ts with 4 mode buttons + Apply button. Update core/index.ts switchMode to handle 4 modes.

### Task 4: Rebuild manipulate - element handles
Create core/manipulate/handles.ts. Render draggable edge handles for padding/margin directly on the selected element. Corner handles for border-radius. Handles are positioned divs in Shadow DOM that track the element's bounding box.

### Task 5: Rebuild manipulate - box model visualization
Create core/manipulate/box-model.ts. Mini diagram in the property panel showing margin > border > padding > content with actual pixel values. Color-coded layers (margin=orange, padding=green, content=blue).

### Task 6: Rebuild manipulate - polished property panel
Rewrite core/manipulate/property-panel.ts. Collapsible sections per control group. Visual color swatches next to hex values. Styled select dropdowns. Shorthand display for padding/margin (4-value visual). State toggle buttons (hover/focus/active/visited).

### Task 7: Rebuild manipulate - state toggles
Create core/manipulate/state-toggle.ts. Buttons that force pseudo-state styles onto the selected element by applying the pseudo-class computed styles as regular CSS rules via the preview engine.

### Task 8: Rebuild annotate - colored markers and rich popups
Rewrite core/annotate/markers.ts. Color-coded markers: fix=red, change=blue, question=yellow, approve=green. Numbered badges. Connecting lines from marker to element. Rich comment popup with element name, path, intent/severity as styled pill buttons.

### Task 9: Rebuild annotate - multi-select feedback and lasso
Rewrite core/annotate/lasso.ts. Visible selection rectangle with element count badge. Selected elements get colored overlays (not just highlight). Freeze indicator with unfreeze button.

### Task 10: Rebuild annotate - Send to Claude button
Add a "Send to Claude" button in annotate mode that pushes all pending annotations to the MCP server with visual confirmation (count sent, green flash).

### Task 11: Rebuild layout - polished palette sidebar
Rewrite core/layout/palette.ts. Categorized with section headers. Searchable filter input at top. Visual icons per component type. Styled to match dark theme.

### Task 12: Rebuild layout - skeleton and snap polish
Rewrite core/layout/skeleton.ts and core/layout/guide-lines.ts. Category-colored backgrounds, rounded corners, labels, resize handles on skeletons. Snap guides with distance labels.

### Task 13: Rebuild layout - Apply Layout button
Add "Apply Layout" button that shows a structured description preview of what will be sent, then sends to MCP server with confirmation.

### Task 14: Apply confirmation UI
Create core/apply-confirmation.ts. Shared component used by all modes. Shows green checkmarks per change, file paths when available, "Sent to Claude" state with fade-out animation. Wire into manipulate, prompt, annotate, and layout modes.

### Task 15: Final build and deploy
Run all tests, build core + adapters, deploy to ~/.claude/improv/, verify in browser.
