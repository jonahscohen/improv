Improv Guide

Visual micro-adjustment tool for front-end developers. Select elements, describe changes in natural language, and see Claude's work reflected back in real time.

Setup

Install

If you have the dotfiles installed, improv is already available. Otherwise:

ampersand --only improv

Activate in a project

Run once per project to wire the script into your dev server:

improv-init

This detects your stack (WordPress, Next.js, Vite, Drupal, static HTML) and adds a <script> tag pointing to the local MCP server. No files are copied - improv is always served fresh from localhost:9223.

Or activate from Claude

Tell Claude:

activate improv

Claude will inject the script into the active browser tab automatically.

The Toolbar

Improv starts as a small circle (I) in the bottom-right corner. Click it to expand the toolbar.

Prompt Mode

Select elements and describe changes in natural language. Claude reads your instructions and makes the code changes.

Manipulate Mode

Click elements to open a property panel with live CSS controls. Scrub values, pick colors, adjust spacing.

Settings

Verbosity level, connection status, marker color (persists per-project), hints toggle, selection labels toggle.

Prompt Mode - The Claude Loop

This is the core workflow. You stay in the browser. Claude stays in the terminal.

1Click an element on the page (or shift-click for multi-select, or lasso-drag)
2Type what you want changed: "make this red", "increase padding to 24px", "swap this font"
3Hit Enter to queue, or Cmd+Enter to send immediately
4Claude picks up your prompt, finds the right source file, makes the change
5Your page reloads with the change applied. A toast confirms what happened.
6Review in the Changes panel. Mark done, reply with adjustments, or revert.

Starting the watch loop

For Claude to pick up your prompts, start the watch loop in a Claude session:

improv watch

Claude enters a persistent loop, polling for new prompts every 60 seconds. When you submit from the browser, Claude acts on it within seconds.

Queueing changes

You can queue multiple changes before sending. Each one gets its own prompt with full element context. The queue count appears in the action bar at the bottom-left.

  • Enter in the prompt input queues the change
  • Cmd+Enter sends immediately
  • Click the send button to flush the entire queue

The Changes Panel

After Claude responds, a Claude button appears at the bottom-left with a badge showing unreviewed changes. Click it to open the Changes panel.

Each entry shows:

  • Status dot: green = completed, blue = needs info, red = failed
  • Summary of what Claude did
  • Files modified
  • Property diffs: padding: 16px → 24px
  • If Claude has a question, it appears as a callout with a reply option

Actions per entry:

  • Done - marks the change as reviewed (dims it)
  • Revert - undoes the CSS preview for that change
  • Reply - opens an inline input to send a follow-up ("actually make it 20px instead")

Use Clear done in the panel header to remove all reviewed entries.

Keyboard Shortcuts

All single-key shortcuts are suppressed when a text input is focused. Only Escape and Cmd combos fire regardless.

Cmd+Shift+. Toggle toolbar expand/collapse
Escape Close panel / exit mode / dismiss prompt
P Toggle Prompt mode
M Toggle Manipulate mode
C Toggle Changes panel
Q Toggle Queue panel
J / K Navigate entries in panels
D Mark focused entry as done
R Reply to focused entry

Marker Color

The marker color is the accent color used throughout improv - active button states, selection highlights, screen glow, badge, focus rings. Change it in Settings. It persists per-project via localStorage.

On light backgrounds (orange, yellow, green), icon colors automatically switch to dark (#1a1a1a) for WCAG AA contrast compliance.

For Developers

Build

# Development (non-minified + source maps)
cd improv && node build.js --dev --core-only

# Production (minified + source maps)
cd improv && node build.js --core-only

# Deploy to MCP server
cp dist/improv-core.js ~/.claude/improv/dist/improv-core.js

Architecture

improv/
  core/           # TypeScript source (~20 files)
    index.ts      # ImprovCore - main orchestrator
    toolbar.ts    # Toolbar pill UI
    icons.ts      # SVG icon builders
    overlay.ts    # Shadow DOM overlay + highlight
    transport.ts  # WebSocket client
    changes-panel.ts  # Claude changes review panel
    prompt/       # Prompt mode (selection, input, queue)
    manipulate/   # Manipulate mode (property panel)
    annotate/     # Annotate mode (lasso, markers)
  server/         # MCP server (TypeScript)
    mcp-tools.ts  # MCP tool definitions
    ws-server.ts  # WebSocket + HTTP server
  build.js        # esbuild bundler
  dist/           # Built output

improv_get_annotationsAnnotations at 4 verbosity levels
improv_clearClear all buffers

Accessibility

  • All buttons have role="button", aria-label, and tabindex="0"
  • Panels use role="dialog" with aria-labelledby
  • Badge counts use aria-live="polite" for screen reader announcements
  • Focus-visible rings follow the marker color via CSS custom property
  • prefers-reduced-motion disables all animations and transitions
  • Minimum 40x40px hit areas on all interactive elements
