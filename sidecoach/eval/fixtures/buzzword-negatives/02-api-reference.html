API Reference

The REST API accepts JSON request bodies and returns JSON responses over HTTPS.

Authentication

Authenticate requests with a bearer token in the Authorization header. Tokens are created in the dashboard under Settings, Keys. Treat tokens as secrets and rotate them if they are exposed.

curl https://api.example.com/v1/messages \
  -H "Authorization: Bearer sk_live_..."

Rate Limits

Each token is limited to 100 requests per second. When you exceed the limit the API returns a 429 status code with a Retry-After header indicating how many seconds to wait before retrying.

Endpoints

Create a message

Send a POST request to /v1/messages with the recipient address, subject, and body. The response includes the message identifier and a delivery status of queued.

Retrieve a message

Send a GET request to /v1/messages/:id to fetch the current status of a message, including timestamps for queued, sent, and delivered events.

List messages

Send a GET request to /v1/messages with optional limit and cursor query parameters. Results are paginated and ordered by creation time, newest first.

Webhooks

Register a webhook URL to receive event notifications. Each delivery is signed with an HMAC signature in the X-Signature header so you can verify the payload originated from the API. Respond with a 200 status code within five seconds to acknowledge receipt.

Error Codes

The API uses conventional HTTP status codes. A 400 indicates a malformed request, a 401 indicates an invalid token, a 404 indicates a missing resource, and a 500 indicates a server error. Error responses include a machine-readable code and a human-readable message.

Versioning

The current version is v1. Breaking changes are released under a new version prefix, and older versions remain supported for at least twelve months after a successor ships.
