Deploy Postgres in 30 seconds

A managed Postgres database with automatic backups, connection pooling, and point-in-time recovery. No servers to manage.

Provision a database

Create a database from the dashboard or the CLI. Each database runs on isolated compute and storage, and you get a connection string the moment it is ready. Branching lets you copy a database for a pull request and discard it when the branch is merged.

Automatic backups

We take a full backup every day and keep write-ahead logs continuously, so you can restore to any second within your retention window. Restores create a new database rather than overwriting the original, so you can inspect the result before switching traffic.

Connection pooling

A built-in pooler sits in front of every database and multiplexes thousands of client connections onto a small number of Postgres backends. This keeps serverless functions from exhausting the connection limit during a traffic spike.

Point-in-time recovery

If a bad migration drops a table, you can branch from a timestamp thirty seconds before the mistake and copy the table back. Recovery is measured in seconds because we restore from the nearest snapshot and replay only the logs since.

Read replicas

Add a read replica in another region with one command. Replicas stay within a second of the primary under normal load and are useful for analytics queries that would otherwise compete with production traffic.

Pricing that scales down

Compute pauses when a database is idle and resumes on the next connection, so a staging database that sees traffic during work hours costs almost nothing overnight. You pay for the storage you use and the compute time you consume, billed by the second.
